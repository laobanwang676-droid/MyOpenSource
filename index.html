嵌入式开发 物联网 边缘计算 RTOS 开源硬件

基于RT-Thread的多串锂电池管理系统(BMS)

一、项目概述

以 STM32F103C8T6 为主控，搭配 BQ76920 电池前端芯片，基于 RT-Thread 实时操作系统开发。采用 ASW-RTE-BSW 三层解耦架构，集成高精度 SOC 估算（OCV + 安时积分 + 温度补偿）、硬件级快速保护、被动均衡控制与 FinSH 命令行交互。搭建 CMake + ARM GCC 构建体系替代 Keil，集成 OpenOCD、ST-Link 与 Cortex-Debug 实现编译烧录与在线调试；同时集成 CAN 通信，支持 BMS 状态上报与电芯数据请求/响应。

二、硬件与技术栈

核心硬件

  • 主控芯片：STM32F103C8T6（ARM Cortex-M3 内核，高性价比工业级主控）
  • 电池前端芯片：BQ76920（多节锂电池电压采集、硬件保护核心芯片）
  • 功能电路：锂电池被动均衡电路、硬件过流 / 短路保护电路
  • 交互接口：UART 串口（FinSH 命令行调试、数据交互）、CAN 总线（BMS 数据通信）

核心技术

  • 操作系统：RT-Thread（多线程调度、IPC 进程间通信）
  • 开发库：STM32 标准库
  • 构建体系：CMake + ARM GCC 交叉编译工具链
  • 烧录调试：OpenOCD + ST-Link + VS Code Cortex-Debug
  • 通信协议：I2C（BQ76920）、UART（FinSH）、CAN（状态上报）
  • 核心机制：互斥锁、信号量、软件定时器、环形缓冲区

三、系统架构与核心机制

ASW-RTE-BSW 三层架构实现完全解耦。系统拆解为采集、保护、分析、均衡、信息交互 5 个独立线程，通过优先级调度并行运行；互斥锁解决 I2C 总线竞态，信号量实现线程同步与中断唤醒；BQ76920 硬件特性 + 外部中断实现毫秒级断电保护；软件定时器精准控制被动均衡时序，提升均衡效率。

四、核心功能

  • 多节锂电状态采集：BQ76920 实时采集单体电压、温度、总电流等参数；
  • 高精度 SOC 估算：OCV 校准初始电量，安时积分 + 温度补偿实时计算剩余电量；
  • 硬件级安全保护：毫秒级响应短路、过流、过压、欠压，快速断电保护；
  • 智能被动均衡：软件定时器控制均衡时序，自动修复单体电压差异；
  • FinSH 命令交互：串口命令行实时查询电池状态、配置保护参数；
  • CAN 通信交互：支持 BMS 状态上报与电芯电压、温度数据请求/响应；
  • ASW-RTE-BSW 三层解耦架构 + RT-Thread 多线程调度，模块化程度高、实时性强；
  • OCV + 安时积分 + 温度补偿融合算法，SOC 估算精度高、稳定性强；
  • CMake + ARM GCC + OpenOCD + Cortex-Debug 开源工具链，替代 Keil 实现全流程开发。

五、关键难点与解决方案

Keil 工程依赖与调试流程不开放 — 搭建 CMake + ARM GCC 构建体系，配合 OpenOCD、ST-Link 与 Cortex-Debug 实现命令行编译、烧录与在线调试，替代 Keil 完成开发闭环。

外设并发与 CPU 开销过高 — 设计硬件中断 + 环形缓冲区 + 信号量机制，线程阻塞等待信号量唤醒，消除 CPU 轮询开销。

多线程数据竞态与资源冲突 — 采用互斥锁保护共享资源，保证多线程并发场景下的数据安全性与业务逻辑稳定。

六、应用场景

电动工具电池包、便携式储能电源、工业锂电模组、小型新能源设备、锂电池教学实训平台。

边缘 AI 的低功耗智能健康监护穿戴设备

一、项目概述

以 STM32U575VGT6 低功耗主控为核心，搭载 FreeRTOS 实时操作系统，打造面向健康监护的可穿戴设备。深度融合边缘 AI 推理、多体征传感器采集、4G 无线通信、GPS 定位、远程 OTA 升级等技术，实现跌倒智能检测、心率血氧监测、紧急告警、定位上报、云端数据互联等功能。采用极致低功耗设计搭配触控界面，兼顾续航与人机体验。

二、硬件平台

  • 主控芯片：STM32U575VGT6，Cortex-M33 内核，具备优异低功耗特性，满足边缘计算运行需求
  • 姿态采集：SC7A20H 三轴加速度计，负责人体运动姿态原始数据采集
  • 体征检测：MAX30102 心率血氧传感器，精准采集人体心率、血氧饱和度参数
  • 定位模块：ATGM336H GPS 定位模组，实现户外精准位置获取
  • 无线通信：Air780E 4G 模块，承担数据上传、短信告警、远程 OTA 通信任务
  • 人机交互：电阻触摸屏 + 液晶显示屏，基于 LVGL 实现可视化界面操作
  • 电源管控：NMOS 低端开关电路，按需通断外设供电；搭载电量检测电路
  • 存储介质：片内 Flash，用于本地配置数据持久化保存

三、核心技术与运行机制

FreeRTOS 任务调度 — 互斥锁与消息队列解决资源竞态，稳定调度体征采集、AI 判别、网络上传、界面刷新等任务。

边缘 AI 跌倒检测 — SC7A20H 采集运动数据，提取统计特征与峰值特征，小型全连接神经网络联合差分尖峰阈值完成行为判别。

物联网数据交互 — ATGM336H + Air780E 通过 MQTT 上传数据至 OneNET 云平台；险情自动发送告警短信并同步位置信息。

LVGL 触控界面 — 多页面交互系统（主页、血氧、GPS、联系人、摔倒提醒、OTA），分散加载技术将核心渲染程序重定位至 SRAM 提升流畅度。

安全 OTA 升级 — HTTP 在线固件更新，MD5 完整性校验，规避固件损坏风险。

本地数据持久化 — 片内 Flash 存储紧急联系人号码，触屏编辑自动保存，断电不丢失。

低功耗与电量管理 — NMOS 低端开关切断闲置外设，一键息屏节能；实时电量检测，低电量自动报警。

四、核心功能

  • 生命体征监测：MAX30102 实时采集心率血氧，本地显示并上传云端存档；
  • 边缘 AI 跌倒检测：多算法联合判定跌倒，识别灵敏、误判率低；
  • 定位与紧急告警：GPS 精准定位，险情后 4G 推送短信与位置信息；
  • 云端数据互通：MQTT 对接 OneNET 云平台，支持远程查看历史数据；
  • 触控人机交互：多场景功能页面自由切换，支持参数自定义配置；
  • 安全 OTA 升级：远程固件更新，MD5 校验保障升级稳定；
  • 智能功耗控制：外设智能断电、息屏节能，延长设备续航；
  • 低功耗穿戴设计：STM32U5 主控 + 智能电源管控，适配日常穿戴场景；
  • 端侧 AI 落地：轻量化神经网络本地识别跌倒，无需依赖云端。

五、关键难点与解决方案

设备功耗偏高 — NMOS 低端开关单独管控外设供电，闲置自动断电；增设息屏唤醒功能，降低待机损耗。

用户配置无法保存 — 读写片内 Flash 完成数据持久化，触屏编辑自动保存，断电重启不丢失。

多任务并发资源争夺 — FreeRTOS 互斥锁管控共享资源，消息队列完成任务间数据传递，规避竞态问题。

图形界面刷新缓慢 — 分散加载将高频渲染函数迁移至 SRAM 运行，大幅提升界面流畅度。

电量状态无法把控 — 增加电量检测电路，界面实时展示电量百分比，临界值主动报警。

六、应用场景

独居老人健康监护、儿童出行防护、慢性病日常监测、个人随身安全防护。

基于FreeRTOS的智能天气时钟

一、项目概述

以 STM32F407VET6 为主控核心，搭载 FreeRTOS 实时操作系统，兼容标准库与 HAL 库双版本开发。深度融合环境感知、网络数据交互、语音智能交互、实时报警与 LVGL 可视化人机交互，实现从数据采集、网络通信到智能响应、本地显示的全流程闭环。

二、硬件与技术栈

核心硬件

  • 主控芯片：STM32F407VET6（高性能 ARM Cortex-M4 内核）
  • 通信模块：ESP32C3 模块（WiFi 联网、网络数据获取）
  • 传感模块：AHT20 I2C 接口温湿度传感器
  • 显示模块：ST7789 LCD 显示屏（SPI 接口 + DMA 传输）
  • 交互模块：SU03T语音识别 / LU6288播报模块、物理按键

核心技术

  • 操作系统：FreeRTOS（任务调度、进程间通信）
  • 驱动框架：LVGL 嵌入式图形库
  • 通信协议：UART 串口、I2C、SPI、DMA
  • 软件兼容：STM32 标准库 / HAL 库双版本支持

三、系统架构与核心机制

FreeRTOS 多任务架构，通过队列与信号量实现数据同步与资源互斥访问。UART 串口与 ESP32 通信，串口中断接收 + 指令校验确保天气数据精准获取；I2C 协议采集温湿度数据；LVGL 驱动 LCD，SPI + DMA 提升屏幕刷新效率；支持按键配网、语音识别、阈值报警等智能化交互。

四、核心功能

  • 智能环境监测：实时采集温湿度，支持自定义阈值声光报警；
  • WiFi 网络交互：按键快速配网，获取公网天气与网络标准时间；
  • 语音智能交互：语音指令识别，播报温湿度、天气、网络时间；
  • LVGL 人机交互：可视化界面展示环境数据、网络状态与报警信息；
  • 高可靠通信：串口指令校验与数据容错，保障模块间通信稳定；
  • 双库兼容：同时支持标准库与 HAL 库，适配不同开发与移植需求；
  • 高效显示：SPI + DMA + LVGL 组合，实现高帧率低功耗界面渲染。

五、关键难点与解决方案

LCD 多任务数据显示竞态 — FreeRTOS 队列统一管理显示数据，避免多任务资源竞争，保证显示流畅与数据准确。

串口数据接收可靠性 — 串口中断接收搭配指令校验机制，确保网络数据解析精准。

多外设资源冲突 — 信号量实现资源互斥访问，FreeRTOS 优先级调度保障核心任务优先执行。

六、应用场景

智能家居环境监测、室内智能终端、嵌入式教学实训、工业环境监控节点。

多模态两轮自平衡系统

一、项目概述

以 STM32F103C8T6 为主控芯片，标准库与 HAL 库双版本开发，打造高性能两轮自平衡智能小车。融合姿态闭环平衡、自主循迹、智能避障、蓝牙远程操控四大核心功能，通过多传感器数据融合与高精度闭环控制算法，实现稳定自主运行与远程交互。

二、硬件平台

  • 主控芯片：STM32F103C8T6（ARM Cortex-M3 内核，高性能低成本）
  • 姿态传感器：MPU6050（六轴陀螺仪 + 加速度计，用于姿态解算）
  • 执行机构：双路 520 直流减速电机，驱动小车运动与平衡
  • 显示模块：0.96 寸 OLED 显示屏，实时显示姿态、运行模式、传感器数据
  • 通信模块：蓝牙串口模块，实现手机 APP 远程无线操控
  • 环境传感器：HC-SR04 超声波模块、五路红外循迹模块
  • 保护机制：倒地检测、过流保护、扶正自启动硬件电路

三、核心技术与控制机制

姿态感知与解算 — 软件 I2C 读取 MPU6050 数据，卡尔曼滤波融合陀螺仪与加速度计，输出高精度实时姿态角。

三级闭环 PID 控制 — 直立环保证动态平衡，速度环实现匀速行驶，转向环完成精准转向，三者协同保障稳定运行。

智能感知控制 — 五路红外 + PD 闭环实现高精度循迹；HC-SR04 超声波实时检测障碍物触发自动避障；蓝牙与 OLED 提供交互与状态显示。

四、核心功能

  • 自平衡控制：三环 PID 实现动态直立平衡，抗干扰能力强；
  • 自主循迹：五路红外 + PD 控制，精准识别黑线轨迹稳定行驶；
  • 智能避障：超声波实时测距，自动绕行障碍物；
  • 蓝牙遥控：手机 APP 远程控制启停、转向、调速；
  • 状态显示：OLED 实时输出姿态角、运行模式、传感器数据；
  • 安全保护：倒地自动停机，扶正后快速恢复平衡；
  • 高精度算法：卡尔曼滤波 + 三环 PID + PD 闭环，控制精度高；
  • 双库兼容：同时支持标准库与 HAL 库，适配不同开发场景。

五、关键难点与解决方案

循迹精度低、行驶抖动 — 采用 PD 闭环控制算法，优化响应速度与循迹平滑度。

MPU6050 姿态解算零漂严重 — 卡尔曼滤波融合多传感器数据，有效抑制零漂，提升平衡稳定性。

多任务控制冲突 — 模块化任务调度，优先级分配核心控制任务，保证系统实时性。

六、应用场景

嵌入式运动控制教学实训、智能小车竞赛、科创 DIY 项目、自主移动机器人入门开发。
